****JENKINS************CI/CD ipeline**********

CI- Continous Interation, CD- Continous Delivery (Deployment)- This is basically a SDLC methodlogy to perform a process.
Pipelines: A step by step process to perform CI/CD.

CI- Continous Interation: Dev will imlement the code and push/move the code in sprints/chunks i.e. on day to day basis, what amnt of wok he does on the code, he'll move it to the IT HUB repository. Later, that'll move to CI server automatically. So, CI server will have 3 imp parts, BUILD, TEST, DEPLOY. So if a code fails with errors after the 3 steps(BUILD, TEST, DEPLOY), Dev will get notified, then he'll rework on the same code and again push it to GIT repo. Once the code is success, dev gets notified as working well then he starts working on other sprint of code.
[So in Short CI = Continus Build + Continous Test]
s
PIPELINE:
Code Implement => move to Version COntrol(GIT) => Build => Unit Test(test small content of code that is moved) => Deploy => Auto Test(we perform functional/regression test) => Prod env Deploy(at our level, i.e on test env) => Measure & Validate(QA does this, if the code works well)

Here, Dev will get notified on every step that if his code is working well or not. i.e. if any issues happen on GIT llevel/Build level/Unit Test level, he gets notified if any eeor comes in above steps.


JENKINS:
Process:-
1. Plan and Code- Plan what to imlement, do the implementation and move to GIT HUB.
2. Build- We build based on tool like MAVEN or GRADLE.
3. Test- Using Selenium or Junit
4. Deploy and Operate- We use chef/puppet, to configure and deploy that code to Machine
5. Monitor- Nagios tool

PLAN						DEPLOY

	       INTWGRATUIION TOOL

Once the code is deployed to GIT, JENKINS will pull the code and will send the code to MAVEN, and again sent it to Test, later will send to chef/puppet for deployment or operate
So, Jenkins serves the task it gets. It automatically performs the task in the pipeline like build, deploy, test etc.

So in easy way to explain:
Dev writes the code => sends/push to GIT => code will be pulled by JENKINS => will send for build and use MAVEN for build process(Build means it compiles the code i.e. create an executable file exe) => After Build JENKINS will again pull that code/file back and will send it to Test(we use selenium) => After test it'll validate the quality assurance part. ==> Once this process is done[i.e. CI, we move to CD], JENKINS will DEPLOY(means to deploy the application on the client side and install it and show how it works to the cient) the code or DELIVER(means application is ready and ready to install)

JENKINS PLUGIN: Plugins are usd to communicate or integrate JENKINS with GIT, Selenium, maven etc to perform diff task.
[Note: GIT will have a dummy/source code and ARTIFACTRY will have a orignal/final code].
