*In which case u will be considering a class as a private constructor. Or in which case we'll be using private constructor in automation framework
A private constructor in Java is used in restricting object creation. It is a special instance constructor used in static member-only classes. If a 
constructor is declared as private, then its objects are only accessible from within the declared class. You cannot access its objects from outside the 
constructor class.
Impact of Private Constructor in Java:-
The private constructors cannot access derived classes from another class. Thus, we have to give a public function, which calls the private constructor. 
In case, the object is not initialized, or we have to send back an instance to the object if it was initialized. This is especially very useful for objects, 
which cannot be initialized. A private constructor is used in the following cases:

The respective classes, which have only static methods and members.
The specific classes, which have only widely used static final members (constants).
To incorporate singletons.
To incorporate the factory methods.

*What is static in Java?
The static keyword is a non-access modifier used for methods and attributes. Static methods/attributes can be accessed without creating an object of a class.
We can apply static keyword with variables, methods, blocks and nested classes. The static keyword belongs to the class than an instance of the class.

The static can be:

Variable (also known as a class variable)
Method (also known as a class method)
Block
Nested class

**Explain about Static Block.
Static blocks are always executed 1st if declared in a Class. We can define multiple static blocks in a class and they get executed sequentially. So, if there
is main mthod and static method and static block, static block executes first then the main method and later other type of methods.
for eg:

static{
syso("this is a static block");
}
public static void test(){
syso("test");
}
public static void main(String[] args){
syso("main");
}
O/P:  this is a static block
	test
	main

**Why the constructors cannot be static?

The constructors in Java can not be static because if the constructors are marked as static, they can not be called from the child class; thus, the child 
class's object will not be created. The program will not be compiled and throw a compile-time error.


**Can constructor be static or final?
No, a constructor can't be made final. A final method cannot be overridden by any subclasses.

** Difference between Comparable and Comparator:

Comparable:
-This allows single sorting sequence in collections. We can sort the collection on the basis of a single element such as id, name, and price.
-This affects the original class that is modified. 
-This provides compareTo() method to sort the elements
-We can sort the list elements of Comparable type by Collections.sort(List) method.

Comparator:
-This allows multiple sorting sequence in collection. We can sort the collection on the basis of a multiple element such as id, name, and price.
-This doesn't affect the original class that is not modified.
-This provides compare() to sort the elements.
-We can sort the list elements of Comparator type by Collections.sort(List, Comparator) method.

** Difference between Array and List:

Array:
-It has fixed size 
-All elements in array must be of same datatype
-Arrays are less flexible as their size cannot be easily changed.

List:
-List are dynamic and can change in size.
-List can accommodate elements of different datatype.
-Lists are more flexible, allowing easy addition or removal of elements.
